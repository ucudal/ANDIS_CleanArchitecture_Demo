1r'''Parse strings using a specification based on the Python format() syntax.
2
3 ``parse()`` is the opposite of ``format()``
4
5The module is set up to only export ``parse()``, ``search()``, ``findall()``,
6and ``with_pattern()`` when ``import \*`` is used:
7
8>>> from parse import *
9
10From there it's a simple thing to parse a string:
11
12.. code-block:: pycon
13
14 >>> parse("It's {}, I love it!", "It's spam, I love it!")
15 <Result ('spam',) {}>
16 >>> _[0]
17 'spam'
18
19Or to search a string for some pattern:
20
21.. code-block:: pycon
22
23 >>> search('Age: {:d}\n', 'Name: Rufus\nAge: 42\nColor: red\n')
24 <Result (42,) {}>
25
26Or find all the occurrences of some pattern in a string:
27
28.. code-block:: pycon
29
30 >>> ''.join(r[0] for r in findall(">{}<", "<p>the <b>bold</b> text</p>"))
31 'the bold text'
32
33If you're going to use the same pattern to match lots of strings you can
34compile it once:
35
36.. code-block:: pycon
37
38 >>> from parse import compile
39 >>> p = compile("It's {}, I love it!")
40 >>> print(p)
41 <Parser "It's {}, I love it!">
42 >>> p.parse("It's spam, I love it!")
43 <Result ('spam',) {}>
44
45("compile" is not exported for ``import *`` usage as it would override the
46built-in ``compile()`` function)
47
48The default behaviour is to match strings case insensitively. You may match with
49case by specifying `case_sensitive=True`:
50
51.. code-block:: pycon
52
53 >>> parse('SPAM', 'spam', case_sensitive=True) is None
54 True

57Format Syntax
58-------------
59
60A basic version of the `Format String Syntax`_ is supported with anonymous
61(fixed-position), named and formatted fields::
62
63 {[field name]:[format spec]}
64
65Field names must be a valid Python identifiers, including dotted names;
66element indexes imply dictionaries (see below for example).
67
68Numbered fields are also not supported: the result of parsing will include
69the parsed fields in the order they are parsed.
70
71The conversion of fields to types other than strings is done based on the
72type in the format specification, which mirrors the ``format()`` behaviour.
73There are no "!" field conversions like ``format()`` has.
74
75Some simple parse() format string examples:
76
77.. code-block:: pycon
78
79 >>> parse("Bring me a {}", "Bring me a shrubbery")
80 <Result ('shrubbery',) {}>
81 >>> r = parse("The {} who {} {}", "The knights who say Ni!")
82 >>> print(r)
83 <Result ('knights', 'say', 'Ni!') {}>
84 >>> print(r.fixed)
85 ('knights', 'say', 'Ni!')
86 >>> print(r[0])
87 knights
88 >>> print(r[1:])
89 ('say', 'Ni!')
90 >>> r = parse("Bring out the holy {item}", "Bring out the holy hand grenade")
91 >>> print(r)
92 <Result () {'item': 'hand grenade'}>
93 >>> print(r.named)
94 {'item': 'hand grenade'}
95 >>> print(r['item'])
96 hand grenade
97 >>> 'item' in r
98 True
99
100Note that `in` only works if you have named fields.
101
102Dotted names and indexes are possible with some limits. Only word identifiers
103are supported (ie. no numeric indexes) and the application must make additional
104sense of the result:
105
106.. code-block:: pycon
107
108 >>> r = parse("Mmm, {food.type}, I love it!", "Mmm, spam, I love it!")
109 >>> print(r)
110 <Result () {'food.type': 'spam'}>
111 >>> print(r.named)
112 {'food.type': 'spam'}
113 >>> print(r['food.type'])
114 spam
115 >>> r = parse("My quest is {quest[name]}", "My quest is to seek the holy grail!")
116 >>> print(r)
117 <Result () {'quest': {'name': 'to seek the holy grail!'}}>
118 >>> print(r['quest'])
119 {'name': 'to seek the holy grail!'}
120 >>> print(r['quest']['name'])
121 to seek the holy grail!
122
123If the text you're matching has braces in it you can match those by including
124a double-brace ``{{`` or ``}}`` in your format string, just like format() does.

127Format Specification
128--------------------
129
130Most often a straight format-less ``{}`` will suffice where a more complex
131format specification might have been used.
132
133Most of `format()`'s `Format Specification Mini-Language`_ is supported:
134
135 [[fill]align][0][width][.precision][type]
136
137The differences between `parse()` and `format()` are:
138
139- The align operators will cause spaces (or specified fill character) to be
140 stripped from the parsed value. The width is not enforced; it just indicates
141 there may be whitespace or "0"s to strip.
142- Numeric parsing will automatically handle a "0b", "0o" or "0x" prefix.
143 That is, the "#" format character is handled automatically by d, b, o
144 and x formats. For "d" any will be accepted, but for the others the correct
145 prefix must be present if at all.
146- Numeric sign is handled automatically.
147- The thousands separator is handled automatically if the "n" type is used.
148- The types supported are a slightly different mix to the format() types. Some
149 format() types come directly over: "d", "n", "%", "f", "e", "b", "o" and "x".
150 In addition some regular expression character group types "D", "w", "W", "s"
151 and "S" are also available.
152- The "e" and "g" types are case-insensitive so there is not need for
153 the "E" or "G" types. The "e" type handles Fortran formatted numbers (no
154 leading 0 before the decimal point).
155
156===== =========================================== ========
157Type Characters Matched Output
158===== =========================================== ========
159l Letters (ASCII) str
160w Letters, numbers and underscore str
161W Not letters, numbers and underscore str
162s Whitespace str
163S Non-whitespace str
164d Digits (effectively integer numbers) int
165D Non-digit str
166n Numbers with thousands separators (, or .) int
167% Percentage (converted to value/100.0) float
168f Fixed-point numbers float
169F Decimal numbers Decimal
170e Floating-point numbers with exponent float
171 e.g. 1.1e-10, NAN (all case insensitive)
172g General number format (either d, f or e) float
173b Binary numbers int
174o Octal numbers int
175x Hexadecimal numbers (lower and upper case) int
176ti ISO 8601 format date/time datetime
177 e.g. 1972-01-20T10:21:36Z ("T" and "Z"
178 optional)
179te RFC2822 e-mail format date/time datetime
180 e.g. Mon, 20 Jan 1972 10:21:36 +1000
181tg Global (day/month) format date/time datetime
182 e.g. 20/1/1972 10:21:36 AM +1:00
183ta US (month/day) format date/time datetime
184 e.g. 1/20/1972 10:21:36 PM +10:30
185tc ctime() format date/time datetime
186 e.g. Sun Sep 16 01:03:52 1973
187th HTTP log format date/time datetime
188 e.g. 21/Nov/2011:00:07:11 +0000
189ts Linux system log format date/time datetime
190 e.g. Nov 9 03:37:44
191tt Time time
192 e.g. 10:21:36 PM -5:30
193===== =========================================== ========
194
195Some examples of typed parsing with ``None`` returned if the typing
196does not match:
197
198.. code-block:: pycon
199
200 >>> parse('Our {:d} {:w} are...', 'Our 3 weapons are...')
201 <Result (3, 'weapons') {}>
202 >>> parse('Our {:d} {:w} are...', 'Our three weapons are...')
203 >>> parse('Meet at {:tg}', 'Meet at 1/2/2011 11:00 PM')
204 <Result (datetime.datetime(2011, 2, 1, 23, 0),) {}>
205
206And messing about with alignment:
207
208.. code-block:: pycon
209
210 >>> parse('with {:>} herring', 'with a herring')
211 <Result ('a',) {}>
212 >>> parse('spam {:^} spam', 'spam lovely spam')
213 <Result ('lovely',) {}>
214
215Note that the "center" alignment does not test to make sure the value is
216centered - it just strips leading and trailing whitespace.
217
218Width and precision may be used to restrict the size of matched text
219from the input. Width specifies a minimum size and precision specifies
220a maximum. For example:
221
222.. code-block:: pycon
223
224 >>> parse('{:.2}{:.2}', 'look') # specifying precision
225 <Result ('lo', 'ok') {}>
226 >>> parse('{:4}{:4}', 'look at that') # specifying width
227 <Result ('look', 'at that') {}>
228 >>> parse('{:4}{:.4}', 'look at that') # specifying both
229 <Result ('look at ', 'that') {}>
230 >>> parse('{:2d}{:2d}', '0440') # parsing two contiguous numbers
231 <Result (4, 40) {}>
232
233Some notes for the date and time types:
234
235- the presence of the time part is optional (including ISO 8601, starting
236 at the "T"). A full datetime object will always be returned; the time
237 will be set to 00:00:00. You may also specify a time without seconds.
238- when a seconds amount is present in the input fractions will be parsed
239 to give microseconds.
240- except in ISO 8601 the day and month digits may be 0-padded.
241- the date separator for the tg and ta formats may be "-" or "/".
242- named months (abbreviations or full names) may be used in the ta and tg
243 formats in place of numeric months.
244- as per RFC 2822 the e-mail format may omit the day (and comma), and the
245 seconds but nothing else.
246- hours greater than 12 will be happily accepted.
247- the AM/PM are optional, and if PM is found then 12 hours will be added
248 to the datetime object's hours amount - even if the hour is greater
249 than 12 (for consistency.)
250- in ISO 8601 the "Z" (UTC) timezone part may be a numeric offset
251- timezones are specified as "+HH:MM" or "-HH:MM". The hour may be one or two
252 digits (0-padded is OK.) Also, the ":" is optional.
253- the timezone is optional in all except the e-mail format (it defaults to
254 UTC.)
255- named timezones are not handled yet.
256
257Note: attempting to match too many datetime fields in a single parse() will
258currently result in a resource allocation issue. A TooManyFields exception
259will be raised in this instance. The current limit is about 15. It is hoped
260that this limit will be removed one day.

268Result and Match Objects
269------------------------
270
271The result of a ``parse()`` and ``search()`` operation is either ``None`` (no match), a
272``Result`` instance or a ``Match`` instance if ``evaluate_result`` is False.
273
274The ``Result`` instance has three attributes:
275
276``fixed``
277 A tuple of the fixed-position, anonymous fields extracted from the input.
278``named``
279 A dictionary of the named fields extracted from the input.
280``spans``
281 A dictionary mapping the names and fixed position indices matched to a
282 2-tuple slice range of where the match occurred in the input.
283 The span does not include any stripped padding (alignment or width).
284
285The ``Match`` instance has one method:
286
287``evaluate_result()``
288 Generates and returns a ``Result`` instance for this ``Match`` object.

292Custom Type Conversions
293-----------------------
294
295If you wish to have matched fields automatically converted to your own type you
296may pass in a dictionary of type conversion information to ``parse()`` and
297``compile()``.
298
299The converter will be passed the field string matched. Whatever it returns
300will be substituted in the ``Result`` instance for that field.
301
302Your custom type conversions may override the builtin types if you supply one
303with the same identifier:
304
305.. code-block:: pycon
306
307 >>> def shouty(string):
308 ... return string.upper()
309 ...
310 >>> parse('{:shouty} world', 'hello world', dict(shouty=shouty))
311 <Result ('HELLO',) {}>
312
313If the type converter has the optional ``pattern`` attribute, it is used as
314regular expression for better pattern matching (instead of the default one):
315
316.. code-block:: pycon
317
318 >>> def parse_number(text):
319 ... return int(text)
320 >>> parse_number.pattern = r'\d+'
321 >>> parse('Answer: {number:Number}', 'Answer: 42', dict(Number=parse_number))
322 <Result () {'number': 42}>
323 >>> _ = parse('Answer: {:Number}', 'Answer: Alice', dict(Number=parse_number))
324 >>> assert _ is None, "MISMATCH"
325
326You can also use the ``with_pattern(pattern)`` decorator to add this
327information to a type converter function:
328
329.. code-block:: pycon
330
331 >>> from parse import with_pattern
332 >>> @with_pattern(r'\d+')
333 ... def parse_number(text):
334 ... return int(text)
335 >>> parse('Answer: {number:Number}', 'Answer: 42', dict(Number=parse_number))
336 <Result () {'number': 42}>
337
338A more complete example of a custom type might be:
339
340.. code-block:: pycon
341
342 >>> yesno_mapping = {
343 ... "yes": True, "no": False,
344 ... "on": True, "off": False,
345 ... "true": True, "false": False,
346 ... }
347 >>> @with_pattern(r"|".join(yesno_mapping))
348 ... def parse_yesno(text):
349 ... return yesno_mapping[text.lower()]
350
351
352If the type converter ``pattern`` uses regex-grouping (with parenthesis),
353you should indicate this by using the optional ``regex_group_count`` parameter
354in the ``with_pattern()`` decorator:
355
356.. code-block:: pycon
357
358 >>> @with_pattern(r'((\d+))', regex_group_count=2)
359 ... def parse_number2(text):
360 ... return int(text)
361 >>> parse('Answer: {:Number2} {:Number2}', 'Answer: 42 43', dict(Number2=parse_number2))
362 <Result (42, 43) {}>
363
364Otherwise, this may cause parsing problems with unnamed/fixed parameters.
365
366
367Potential Gotchas
368-----------------
369
370``parse()`` will always match the shortest text necessary (from left to right)
371to fulfil the parse pattern, so for example:
372
373
374.. code-block:: pycon
375
376 >>> pattern = '{dir1}/{dir2}'
377 >>> data = 'root/parent/subdir'
378 >>> sorted(parse(pattern, data).named.items())
379 [('dir1', 'root'), ('dir2', 'parent/subdir')]
380
381So, even though `{'dir1': 'root/parent', 'dir2': 'subdir'}` would also fit
382the pattern, the actual match represents the shortest successful match for
383``dir1``.

487def with_pattern(pattern, regex_group_count=None):
488 r"""Attach a regular expression pattern matcher to a custom type converter
489 function.
490
491 This annotates the type converter with the :attr:`pattern` attribute.
492
493 EXAMPLE:
494 >>> import parse
495 >>> @parse.with_pattern(r"\d+")
496 ... def parse_number(text):
497 ... return int(text)
498
499 is equivalent to:
500
501 >>> def parse_number(text):
502 ... return int(text)
503 >>> parse_number.pattern = r"\d+"
504
505 :param pattern: regular expression pattern (as text)
506 :param regex_group_count: Indicates how many regex-groups are in pattern.
507 :return: wrapped function
508 """
509
510 def decorator(func):
511 func.pattern = pattern
512 func.regex_group_count = regex_group_count
513 return func
514
515 return decorator

518class int_convert:
519 """Convert a string to an integer.
520
521 The string may start with a sign.
522
523 It may be of a base other than 2, 8, 10 or 16.
524
525 If base isn't specified, it will be detected automatically based
526 on a string format. When string starts with a base indicator, 0#nnnn,
527 it overrides the default base of 10.
528
529 It may also have other non-numeric characters that we can ignore.
530 """
531
532 CHARS = '0123456789abcdefghijklmnopqrstuvwxyz'
533
534 def __init__(self, base=None):
535 self.base = base
536
537 def __call__(self, string, match):
538 if string[0] == '-':
539 sign = -1
540 number_start = 1
541 elif string[0] == '+':
542 sign = 1
543 number_start = 1
544 else:
545 sign = 1
546 number_start = 0
547
548 base = self.base
549 # If base wasn't specified, detect it automatically
550 if base is None:
551
552 # Assume decimal number, unless different base is detected
553 base = 10
554
555 # For number formats starting with 0b, 0o, 0x, use corresponding base ...
556 if string[number_start] == '0' and len(string) - number_start > 2:
557 if string[number_start + 1] in 'bB':
558 base = 2
559 elif string[number_start + 1] in 'oO':
560 base = 8
561 elif string[number_start + 1] in 'xX':
562 base = 16
563
564 chars = int_convert.CHARS[:base]
565 string = re.sub('[^%s]' % chars, '', string.lower())
566 return sign * int(string, base)

585class FixedTzOffset(tzinfo):
586 """Fixed offset in minutes east from UTC."""
587
588 ZERO = timedelta(0)
589
590 def __init__(self, offset, name):
591 self._offset = timedelta(minutes=offset)
592 self._name = name
593
594 def __repr__(self):
595 return '<%s %s %s>' % (self.__class__.__name__, self._name, self._offset)
596
597 def utcoffset(self, dt):
598 return self._offset
599
600 def tzname(self, dt):
601 return self._name
602
603 def dst(self, dt):
604 return self.ZERO
605
606 def __eq__(self, other):
607 if not isinstance(other, FixedTzOffset):
608 return False
609 return self._name == other._name and self._offset == other._offset

645def date_convert(
646 string,
647 match,
648 ymd=None,
649 mdy=None,
650 dmy=None,
651 d_m_y=None,
652 hms=None,
653 am=None,
654 tz=None,
655 mm=None,
656 dd=None,
657):
658 """Convert the incoming string containing some date / time info into a
659 datetime instance.
660 """
661 groups = match.groups()
662 time_only = False
663 if mm and dd:
664 y = datetime.today().year
665 m = groups[mm]
666 d = groups[dd]
667 elif ymd is not None:
668 y, m, d = re.split(r'[-/\s]', groups[ymd])
669 elif mdy is not None:
670 m, d, y = re.split(r'[-/\s]', groups[mdy])
671 elif dmy is not None:
672 d, m, y = re.split(r'[-/\s]', groups[dmy])
673 elif d_m_y is not None:
674 d, m, y = d_m_y
675 d = groups[d]
676 m = groups[m]
677 y = groups[y]
678 else:
679 time_only = True
680
681 H = M = S = u = 0
682 if hms is not None and groups[hms]:
683 t = groups[hms].split(':')
684 if len(t) == 2:
685 H, M = t
686 else:
687 H, M, S = t
688 if '.' in S:
689 S, u = S.split('.')
690 u = int(float('.' + u) * 1000000)
691 S = int(S)
692 H = int(H)
693 M = int(M)
694
695 if am is not None:
696 am = groups[am]
697 if am:
698 am = am.strip()
699 if am == 'AM' and H == 12:
700 # correction for "12" hour functioning as "0" hour: 12:15 AM = 00:15 by 24 hr clock
701 H -= 12
702 elif am == 'PM' and H == 12:
703 # no correction needed: 12PM is midday, 12:00 by 24 hour clock
704 pass
705 elif am == 'PM':
706 H += 12
707
708 if tz is not None:
709 tz = groups[tz]
710 if tz == 'Z':
711 tz = FixedTzOffset(0, 'UTC')
712 elif tz:
713 tz = tz.strip()
714 if tz.isupper():
715 # TODO use the awesome python TZ module?
716 pass
717 else:
718 sign = tz[0]
719 if ':' in tz:
720 tzh, tzm = tz[1:].split(':')
721 elif len(tz) == 4: # 'snnn'
722 tzh, tzm = tz[1], tz[2:4]
723 else:
724 tzh, tzm = tz[1:3], tz[3:5]
725 offset = int(tzm) + int(tzh) * 60
726 if sign == '-':
727 offset = -offset
728 tz = FixedTzOffset(offset, tz)
729
730 if time_only:
731 d = time(H, M, S, u, tzinfo=tz)
732 else:
733 y = int(y)
734 if m.isdigit():
735 m = int(m)
736 else:
737 m = MONTHS_MAP[m]
738 d = int(d)
739 d = datetime(y, m, d, H, M, S, u, tzinfo=tz)
740
741 return d

760def extract_format(format, extra_types):
761 """Pull apart the format [[fill]align][0][width][.precision][type]"""
762 fill = align = None
763 if format[0] in '<>=^':
764 align = format[0]
765 format = format[1:]
766 elif len(format) > 1 and format[1] in '<>=^':
767 fill = format[0]
768 align = format[1]
769 format = format[2:]
770
771 zero = False
772 if format and format[0] == '0':
773 zero = True
774 format = format[1:]
775
776 width = ''
777 while format:
778 if not format[0].isdigit():
779 break
780 width += format[0]
781 format = format[1:]
782
783 if format.startswith('.'):
784 # Precision isn't needed but we need to capture it so that
785 # the ValueError isn't raised.
786 format = format[1:] # drop the '.'
787 precision = ''
788 while format:
789 if not format[0].isdigit():
790 break
791 precision += format[0]
792 format = format[1:]
793
794 # the rest is the type, if present
795 type = format
796 if type and type not in ALLOWED_TYPES and type not in extra_types:
797 raise ValueError('format spec %r not recognised' % type)
798
799 return locals()

941 def _expand_named_fields(self, named_fields):
942 result = {}
943 for field, value in named_fields.items():
944 # split 'aaa[bbb][ccc]...' into 'aaa' and '[bbb][ccc]...'
945 basename, subkeys = re.match(r'([^\‍[]+)(.*)', field).groups()
946
947 # create nested dictionaries {'aaa': {'bbb': {'ccc': ...}}}
948 d = result
949 k = basename
950
951 if subkeys:
952 for subkey in re.findall(r'\‍[[^\‍]]+\‍]', subkeys):
953 d = d.setdefault(k, {})
954 k = subkey[1:-1]
955
956 # assign the value to the last key
957 d[k] = value
958
959 return result

961 def evaluate_result(self, m):
962 '''Generate a Result instance for the given regex match object'''
963 # ok, figure the fixed fields we've pulled out and type convert them
964 fixed_fields = list(m.groups())
965 for n in self._fixed_fields:
966 if n in self._type_conversions:
967 fixed_fields[n] = self._type_conversions[n](fixed_fields[n], m)
968 fixed_fields = tuple(fixed_fields[n] for n in self._fixed_fields)
969
970 # grab the named fields, converting where requested
971 groupdict = m.groupdict()
972 named_fields = {}
973 name_map = {}
974 for k in self._named_fields:
975 korig = self._group_to_name_map[k]
976 name_map[korig] = k
977 if k in self._type_conversions:
978 value = self._type_conversions[k](groupdict[k], m)
979 else:
980 value = groupdict[k]
981
982 named_fields[korig] = value
983
984 # now figure the match spans
985 spans = dict((n, m.span(name_map[n])) for n in named_fields)
986 spans.update((i, m.span(n + 1)) for i, n in enumerate(self._fixed_fields))
987
988 # and that's our result
989 return Result(fixed_fields, self._expand_named_fields(named_fields), spans)

994 def _generate_expression(self):
995 # turn my _format attribute into the _expression attribute
996 e = []
997 for part in PARSE_RE.split(self._format):
998 if not part:
999 continue
1000 elif part == '{{':
1001 e.append(r'\{')
1002 elif part == '}}':
1003 e.append(r'\}')
1004 elif part[0] == '{' and part[-1] == '}':
1005 # this will be a braces-delimited field to handle
1006 e.append(self._handle_field(part))
1007 else:
1008 # just some text to match
1009 e.append(REGEX_SAFETY.sub(self._regex_replace, part))
1010 return ''.join(e)

1012 def _to_group_name(self, field):
1013 # return a version of field which can be used as capture group, even
1014 # though it might contain '.'
1015 group = field.replace('.', '_').replace('[', '_').replace(']', '_')
1016
1017 # make sure we don't collide ("a.b" colliding with "a_b")
1018 n = 1
1019 while group in self._group_to_name_map:
1020 n += 1
1021 if '.' in field:
1022 group = field.replace('.', '_' * n)
1023 elif '_' in field:
1024 group = field.replace('_', '_' * n)
1025 else:
1026 raise KeyError('duplicated group name %r' % (field,))
1027
1028 # save off the mapping
1029 self._group_to_name_map[group] = field
1030 self._name_to_group_map[field] = group
1031 return group

1033 def _handle_field(self, field):
1034 # first: lose the braces
1035 field = field[1:-1]
1036
1037 # now figure whether this is an anonymous or named field, and whether
1038 # there's any format specification
1039 format = ''
1040
1041 if ':' in field:
1042 name, format = field.split(':')
1043 else:
1044 name = field
1045
1046 # This *should* be more flexible, but parsing complicated structures
1047 # out of the string is hard (and not necessarily useful) ... and I'm
1048 # being lazy. So for now `identifier` is "anything starting with a
1049 # letter" and digit args don't get attribute or element stuff.
1050 if name and name[0].isalpha():
1051 if name in self._name_to_group_map:
1052 if self._name_types[name] != format:
1053 raise RepeatedNameError(
1054 'field type %r for field "%s" '
1055 'does not match previous seen type %r'
1056 % (format, name, self._name_types[name])
1057 )
1058 group = self._name_to_group_map[name]
1059 # match previously-seen value
1060 return r'(?P=%s)' % group
1061 else:
1062 group = self._to_group_name(name)
1063 self._name_types[name] = format
1064 self._named_fields.append(group)
1065 # this will become a group, which must not contain dots
1066 wrap = r'(?P<%s>%%s)' % group
1067 else:
1068 self._fixed_fields.append(self._group_index)
1069 wrap = r'(%s)'
1070 group = self._group_index
1071
1072 # simplest case: no type specifier ({} or {name})
1073 if not format:
1074 self._group_index += 1
1075 return wrap % r'.+?'

1077 # decode the format specification
1078 format = extract_format(format, self._extra_types)
1079
1080 # figure type conversions, if any
1081 type = format['type']
1082 is_numeric = type and type in 'n%fegdobx'
1083 if type in self._extra_types:
1084 type_converter = self._extra_types[type]
1085 s = getattr(type_converter, 'pattern', r'.+?')

1087 if regex_group_count is None:

1098 self._group_index += 1
1099 elif type == 'o':
1100 s = r'(0[oO])?[0-7]+'
1101 self._type_conversions[group] = int_convert(8)
1102 self._group_index += 1
1103 elif type == 'x':
1104 s = r'(0[xX])?[0-9a-fA-F]+'
1105 self._type_conversions[group] = int_convert(16)
1106 self._group_index += 1
1107 elif type == '%':
1108 s = r'\d+(\.\d+)?%'
1109 self._group_index += 1
1110 self._type_conversions[group] = percentage
1111 elif type == 'f':
1112 s = r'\d*\.\d+'
1113 self._type_conversions[group] = convert_first(float)
1114 elif type == 'F':
1115 s = r'\d*\.\d+'

1120 elif type == 'g':
1121 s = r'\d+(\.\d+)?([eE][-+]?\d+)?|nan|NAN|[-+]?inf|[-+]?INF'

1261

1263 if align == '<':

1265 elif align == '>':
1266 s = '%s*%s' % (fill, s)
1267 elif align == '^':

1348def parse(format, string, extra_types=None, evaluate_result=True, case_sensitive=False):
1349 """Using "format" attempt to pull values from "string".
1350
1351 The format must match the string contents exactly. If the value
1352 you're looking for is instead just a part of the string use
1353 search().
1354
1355 If ``evaluate_result`` is True the return value will be an Result instance with two attributes:
1356
1357 .fixed - tuple of fixed-position values from the string
1358 .named - dict of named values from the string
1359
1360 If ``evaluate_result`` is False the return value will be a Match instance with one method:
1361
1362 .evaluate_result() - This will return a Result instance like you would get
1363 with ``evaluate_result`` set to True
1364
1365 The default behaviour is to match strings case insensitively. You may match with
1366 case by specifying case_sensitive=True.
1367
1368 If the format is invalid a ValueError will be raised.
1369
1370 See the module documentation for the use of "extra_types".
1371
